    @theredspace/spp-client

    @theREDspace/spp-client api module

    A lightweight, TypeScript-first client for the OpenAir SuiteProjects Pro (SPP) XML API. Provides SPPClient and easy-to-use methods (list, add, batchAdd, batchUpdate, delete, etc.) for common business objects.

    Installation

    Install the latest version:

    npm install @theredspace/spp-client@latest
    # or
    yarn add @theredspace/spp-client@latest

    Authentication

    Before making API calls, you need OAuth tokens. Follow these steps:

    1. Get the authorization URL and redirect the user

      import { SPPClient } from "@theREDspace/spp-api-client";
      const authClient = new SPPClient({
        sppUrl: process.env.SPP_URL,
        clientId: process.env.SPP_CLIENT_ID,
        clientSecret: process.env.SPP_CLIENT_SECRET,
        callbackUrl: process.env.SPP_CALLBACK_URL,
      });
      const authorizationUrl = authClient.getAuthUrl();
      console.log("Visit this URL to authorize:", authorizationUrl);

    2. Exchange the authorization code for tokens

      After the user grants access, your app receives a code query parameter at your callback URL. Exchange it for tokens:

      curl -X POST "${SPP_URL}/login/oauth2/v1/token" \
        -u "${CLIENT_ID}:${CLIENT_SECRET}" \
        -d "grant_type=authorization_code&code=AUTH_CODE&redirect_uri=${CALLBACK_URL}"

      Response:

      {
        "access_token": "<ACCESS_TOKEN>",
        "refresh_token": "<REFRESH_TOKEN>"
      }

    3. Initialize client with tokens

      const client = new SPPClient({
        sppUrl: process.env.SPP_URL,
        clientId: process.env.SPP_CLIENT_ID,
        clientSecret: process.env.SPP_CLIENT_SECRET,
        callbackUrl: process.env.SPP_CALLBACK_URL,
        accessToken: "<ACCESS_TOKEN>",
        refreshToken: "<REFRESH_TOKEN>",
        onRefresh: async ({ access_token, refresh_token }) => {
          // persist new tokens
        },
      });

    Quick Start

    1. Configure environment (either via constructor or env vars):

      export SPP_URL="https://your.openair.com/suiteprojects/etc/etc/"
      export SPP_CLIENT_ID="<your_client_id>"
      export SPP_CLIENT_SECRET="<your_client_secret>"
      export SPP_CALLBACK_URL="<your_callback_url>"

    2. Initialize client:

      import { SPPClient } from "@theREDspace/spp-api-client";

      const client = new SPPClient({
        sppUrl: process.env.SPP_URL,
        clientId: process.env.SPP_CLIENT_ID,
        clientSecret: process.env.SPP_CLIENT_SECRET,
        callbackUrl: process.env.SPP_CALLBACK_URL,
        onRefresh: async ({ access_token, refresh_token }) => {
          // persist tokens securely
        },
      });

    3. Use core methods:

      // Fetch records
      const customers = await client.list("Customer", { country: "US" });

      // Add single or multiple
      await client.add("Invoice", { customerid: 1, total: 100 });
      await client.batchAdd("Invoice", [
        /* array of invoices */
      ]);

      // Update
      await client.update("Invoice", "123", { total: 150 });
      await client.batchUpdate("Invoice", [
        { id: "123", changes: { notes: "Updated" } },
        /* ... */
      ]);

      // Delete one or many
      await client.delete("Invoice", "123");
      await client.delete("Invoice", ["123", "456"]);

    4. Full example (batch operations):

      import { getSPPClient } from "./utils/client";

      (async () => {
        const client = await getSPPClient();

        // 1) Batch add
        const invoices = await client.batchAdd("Invoice", [
          { customerid: 1, total: 100, externalid: "test1" },
          { customerid: 1, total: 200, externalid: "test2" },
        ]);

        // 2) Batch update
        const updates = invoices.map((i) => ({
          id: i.id.toString(),
          changes: { notes: "Batch updated" },
        }));
        await client.batchUpdate("Invoice", updates);

        // 3) Batch delete
        await client.delete(
          "Invoice",
          invoices.map((i) => i.id.toString())
        );
      })();

    Error Handling

    All methods throw typed errors:

    • SPPAuthError (authentication)
    • SPPRequestError (network/HTTP)
    • SPPResponseError (parsing)
    • SPPBusinessError (API-level)

    Wrap calls in try/catch to handle accordingly.
